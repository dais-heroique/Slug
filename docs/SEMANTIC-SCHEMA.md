# Doc 1 — Slug Semantic Schema Specification

**Milestone:** M0, M2  
**Status:** Final draft

-----

## 1. Overview

The **SlugDocument** (SlugDoc) is the wire representation of a UI surface’s semantic state. It is a typed, ordered tree of **SlugNode** objects. The compositor emits a full SlugDoc on first connection, then emits compressed **SlugDelta** frames on every Wayland frame commit. The session daemon materialises the current tree in memory; agents never receive a full tree after the initial snapshot.

-----

## 2. SlugNode Schema

Every node in the tree conforms to this schema. All fields except `ref` and `role` are optional (omit ≡ unknown/not applicable).

```typescript
interface SlugNode {
  // Identity
  ref:           string;          // 128-bit ULID, stable across redraws. See §4.
  parent_ref:    string | null;   // null only for root node
  child_refs:    string[];        // ordered

  // Classification
  role:          SlugRole;        // See §3.1
  states:        SlugState[];     // See §3.2
  
  // Human-readable labels (at least one MUST be present for interactive nodes)
  name:          string | null;   // accessible name (AT-SPI2: accessible_name)
  description:   string | null;   // tooltip / aria-describedby equivalent
  placeholder:   string | null;   // input placeholder text

  // Geometry (compositor-space pixels, unscaled)
  bounds: {
    x: number; y: number;
    width: number; height: number;
  } | null;

  // Value semantics
  value:         string | null;   // current value as string
  value_min:     number | null;   // for sliders, spinners
  value_max:     number | null;
  value_step:    number | null;
  options:       SlugOption[] | null;  // for combo, list, radio groups

  // Text content (for STATIC_TEXT, PARAGRAPH, HEADING nodes)
  text_content:  string | null;
  heading_level: 1|2|3|4|5|6 | null;

  // Interaction affordances
  actions:       SlugAction[];    // what the agent can do to this node
  keyboard_shortcut: string | null;  // e.g. "Ctrl+S"
  
  // Application context
  app_id:        string;          // reverse-DNS, e.g. "org.gnome.Files"
  window_id:     string;          // opaque window identifier
  surface_id:    string;          // Wayland wl_surface id (decimal)

  // Validation / error state (richer than AT-SPI2)
  validation: {
    state:   "valid" | "invalid" | "pending" | null;
    message: string | null;
  } | null;

  // Extension bag — typed key/value for toolkit-specific metadata
  // Keys are reverse-DNS strings. Slug core never reads these.
  extensions:    Record<string, string> | null;
}

interface SlugOption {
  value:    string;
  label:    string;
  selected: boolean;
}

interface SlugAction {
  id:      string;    // stable action id, e.g. "activate", "expand", "set_value"
  label:   string;    // human/agent-readable description
}
```

### 2.1 Design decisions

**Why ULID for ref?** ULIDs are time-sortable, URL-safe, and 128-bit — low collision probability without coordination. They are generated by the compositor at widget creation time and embedded in the Wayland protocol message. The session daemon stores them verbatim.

**Why bounds in compositor-space unscaled pixels?** The agent should not need to know the display scale factor to describe geometry. If the agent needs to click at a screen position, the MCP `pointer_click` tool accepts `ref` not coordinates; bounds are informational only.

**Why `extensions` instead of a flat bag of arbitrary fields?** Keeping the core schema closed against arbitrary keys prevents schema drift. Toolkit-specific data (GTK `accessible-role-description`, Qt `accessibleDescription`) is funnelled through extensions and ignored by Slug core.

## 4. Stable-Ref Scheme

### 4.1 Generation

The compositor generates a `slug_ref` (ULID v4, 128-bit) for each `wl_surface`-backed widget at the moment it is registered in the accessibility tree. The ULID timestamp component uses the compositor clock (`CLOCK_MONOTONIC`). The ref is attached to the Wayland object as a private property and sent in the `slug_semantic_v1::node_created` event.

### 4.2 Stability guarantees

A ref is **stable** (unchanged) across:

- Widget redraws triggered by style/theme changes
- Scroll position changes
- Window resize / relayout
- Application internal state changes that do not destroy and recreate the widget

A ref is **recycled** (can be reused after a 60-second embargo) when:

- The widget is explicitly destroyed (`slug_semantic_v1::node_destroyed` event)
- The application exits
- The Wayland surface is unmapped

The 60-second embargo prevents the agent from acting on a stale ref from a destroyed widget that was replaced by a new widget that coincidentally got the same ref.

### 4.3 Ref format

```
01JXYZ...  (26-char Crockford Base32 string)
Example: 01JWQP7R3C1X8M4GNBSD6V0F2K
```

Refs are opaque to the agent. The agent MUST NOT parse or construct refs.

## 5. Delta / Event Model

### 5.1 Frame lifecycle

```
Wayland frame commit
       │
       ▼
Compositor accessibility hook
       │
       ├─► [no changes] ──► nothing emitted
       │
       └─► [changes exist]
              │
              ▼
         Diff engine computes SlugDelta
              │
              ▼
         Session daemon receives SlugDelta over Wayland socket
              │
              ▼
         Session daemon updates in-memory tree
              │
              ├─► Emits D-Bus signal: org.slug.Semantic.TreeChanged
              │
              └─► MCP server notifies subscribed agent via SSE
```

### 5.2 SlugDelta wire format

```typescript
interface SlugDelta {
  frame_id:     number;    // monotonically increasing per surface
  surface_id:   string;
  timestamp_us: number;    // microseconds, CLOCK_MONOTONIC

  created:   SlugNode[];           // new nodes
  updated:   SlugNodePatch[];      // only changed fields
  destroyed: string[];             // refs of destroyed nodes
  reordered: SlugReorder[];        // child_refs changed
  focus_changed: string | null;    // ref of newly focused node, or null
}

interface SlugNodePatch {
  ref:    string;
  // Only fields that changed are present. Absence = no change.
  states?:       SlugState[];
  value?:        string | null;
  name?:         string | null;
  bounds?:       { x:number; y:number; width:number; height:number } | null;
  validation?:   { state: string; message: string | null } | null;
  // ... any SlugNode field
}

interface SlugReorder {
  parent_ref:  string;
  child_refs:  string[];    // complete new ordered list
}
```

### 5.4 Initial snapshot

On first connection, the session daemon sends a `SlugSnapshot` message containing the complete current tree as a flat array of `SlugNode` objects in breadth-first order. The agent reconstructs the tree using `parent_ref` links. Snapshot delivery is gated behind a session daemon capability check; the agent must present a valid capability token.
